# 文档编写指南 - 自动识别规则

## 📋 标题层级规则

### 一级标题（H1）`#` - 主分类标题
- **识别规则**: 所有以 `# ` 开头的标题
- **显示位置**: 侧边栏最外层，带图标的大标题
- **图标分配**: 按出现顺序自动分配（Rocket, BookOpen, LineChart, User, HelpCircle, Wrench...）
- **示例**:
  ```markdown
  # 快速入门
  # 核心功能模块详解
  # 理解您的核心指标
  ```

### 二级标题（H2）`##` - 可点击的子章节
- **识别规则**: 所有以 `## ` 开头的标题
- **显示位置**: 侧边栏中，显示在对应主分类下方
- **锚点生成**: 自动生成 ID（标题转小写，空格替换为连字符，特殊字符移除）
- **示例**:
  ```markdown
  ## 注册账号
  ## 总览（Overview）
  ## 可见度分析（Visibility）
  ```

### 三级标题（H3）`###` - 文档内容子章节
- **识别规则**: 所有以 `### ` 开头的标题
- **显示位置**: 仅出现在文档内容中，**不会出现在侧边栏**
- **用途**: 用于组织文档内容的详细说明
- **示例**:
  ```markdown
  ### 核心KPI指标卡
  ### 品牌影响力趋势图
  ```

### 四级及以下标题（H4-H6）
- **识别规则**: `#### `, `##### `, `###### `
- **显示位置**: 仅出现在文档内容中，**不会出现在侧边栏**
- **用途**: 更细粒度的内容组织

---

## 📝 文档结构模板

```mdx
---
title: 文档标题
description: 文档描述
---

# 主分类1（H1 - 会出现在侧边栏，带图标）

这是主分类的说明文字。

## 子章节1（H2 - 会出现在侧边栏，可点击）

这是子章节的内容...

### 详细说明1（H3 - 不会出现在侧边栏）

详细内容...

### 详细说明2（H3 - 不会出现在侧边栏）

详细内容...

## 子章节2（H2 - 会出现在侧边栏，可点击）

这是另一个子章节...

---

# 主分类2（H1 - 会出现在侧边栏，带图标）

这是另一个主分类。

## 子章节1（H2 - 会出现在侧边栏，可点击）

详细内容...

## 子章节2（H2 - 会出现在侧边栏，可点击）

详细内容...

---

# 主分类3（H1 - 会出现在侧边栏，带图标）

这是没有子章节的主分类（主分类本身可以作为链接）。

详细内容...
```

## 🎯 识别规则总结

| 标题层级 | Markdown | 侧边栏显示 | 图标 | 锚点链接 |
|---------|----------|-----------|------|---------|
| H1 | `# 标题` | ✅ 主分类（大标题） | ✅ 自动分配 | ✅ 自动生成 |
| H2 | `## 标题` | ✅ 子章节（可点击） | ❌ | ✅ 自动生成 |
| H3 | `### 标题` | ❌ 仅文档内容 | ❌ | ✅ 自动生成 |
| H4+ | `#### 标题` | ❌ 仅文档内容 | ❌ | ✅ 自动生成 |

## ✍️ 编写示例

### ✅ 正确示例

```mdx
# 快速入门

## 欢迎来到 ximu

内容...

## 注册账号

### 步骤1：访问官网

详细步骤...

### 步骤2：填写信息

详细步骤...

---

# 核心功能模块详解

## 总览（Overview）

内容...

## 可见度分析（Visibility）

内容...
```

**结果**:
- 侧边栏会显示：
  - 🚀 **快速入门**
    - 欢迎来到 ximu
    - 注册账号
  - 📖 **核心功能模块详解**
    - 总览（Overview）
    - 可见度分析（Visibility）

### ❌ 错误示例

```mdx
# 快速入门
## 欢迎来到 ximu
# 核心功能模块详解  ❌ 错误：H1 之间没有用 --- 分隔
## 总览
```

**问题**: H1 之间应该用 `---` 分隔，提高可读性（虽然系统也能识别）

## 🔧 自动提取流程

1. **读取文档**: 系统读取 `content/docs/index.mdx`
2. **解析 Front Matter**: 提取 title 和 description
3. **提取 H1 标题**: 识别所有主分类
4. **提取 H2 标题**: 识别每个主分类下的子章节
5. **生成锚点 ID**: 自动为每个标题生成唯一 ID
6. **分配图标**: 按顺序分配图标
7. **生成导航**: 自动生成侧边栏导航结构

---

## 📌 编写建议

1. **主分类之间用 `---` 分隔**，提高可读性
2. **每个主分类下至少有一个 H2 子章节**，如果没有，主分类本身可以作为链接
3. **标题命名要清晰**，便于用户理解
4. **保持层级一致**，不要跳级使用标题（例如：H1 后直接 H3）
5. **使用中文标题**，系统会自动处理锚点 ID 生成
