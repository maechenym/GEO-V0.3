# 文档编写模板

## 标题层级规则

### 一级标题（H1）- 主分类标题
- **格式**: `# 标题名称`
- **用途**: 作为侧边栏的主分类（带图标的大标题）
- **示例**: `# 快速入门`、`# 核心功能模块详解`

### 二级标题（H2）- 可点击的子章节
- **格式**: `## 标题名称`
- **用途**: 作为侧边栏中可点击的子项目
- **示例**: `## 注册账号`、`## 总览（Overview）`

### 三级标题（H3）- 文档内容中的子章节
- **格式**: `### 标题名称`
- **用途**: 文档内容中的细分章节，不会出现在侧边栏
- **示例**: `### 核心KPI指标卡`、`### 品牌影响力趋势图`

### 四级标题（H4-H6）- 更细粒度的内容
- **格式**: `#### 标题名称`、`##### 标题名称`、`###### 标题名称`
- **用途**: 文档内容中的详细说明，不会出现在侧边栏

## 自动识别规则

### 主分类识别
- **规则**: 所有 `# 标题`（一级标题）会被识别为主分类
- **图标分配**: 按顺序自动分配图标（Rocket, BookOpen, LineChart, User, HelpCircle, Wrench）
- **位置**: 显示在侧边栏的最外层，带图标

### 子章节识别
- **规则**: 所有 `## 标题`（二级标题）会被识别为可点击的子章节
- **位置**: 显示在对应主分类下方
- **锚点**: 自动生成 ID（标题转小写，空格替换为连字符）

### 内容章节识别
- **规则**: `### 标题` 及以下层级不会出现在侧边栏
- **用途**: 仅用于文档内容组织

---

## 编写建议

1. **主分类（H1）之间用 `---` 分隔**，提高可读性
2. **每个主分类下至少有一个子章节（H2）**，如果没有子章节，主分类本身可以作为链接
3. **标题命名要清晰**，便于用户理解
4. **保持层级一致**，不要跳级使用标题

## 自动提取工具

系统会自动：
1. 提取所有 H1 标题作为主分类
2. 提取每个 H1 下的 H2 标题作为子章节
3. 为每个标题生成锚点 ID
4. 自动分配图标（按顺序）
5. 生成侧边栏导航结构

你只需要按照这个模板编写文档，系统会自动识别并生成导航！
